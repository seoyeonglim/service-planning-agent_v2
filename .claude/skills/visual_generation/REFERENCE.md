# Visual Generation — 상세 참조 문서

모든 시각 산출물은 **HTML / Mermaid**로 직접 작성한다. 외부 이미지 생성 API(예: Gemini)는 사용하지 않는다.

> 이 문서는 시각 산출물 **세부 규칙의 정본**이다 — 특히 **콜아웃 색상표**(Type C)와 **와이어프레임·도해 파일명 규칙**은 여기가 원본이며, 다른 문서(스킬 11·SKILL.md·CLAUDE.md)와 다르면 이 문서가 우선한다.

## Type A — Mermaid 플로우 다이어그램 구조

```
[다이어그램 제목]: [서비스명/기능명] 흐름
[시작 ~ 종료 범위]: [시작 조건] → [종료 조건]
[주요 단계 (최대 10개)]:
  1. [단계명]: [간단한 설명]
  2. [분기점]: [조건A] → [결과A] / [조건B] → [결과B]
[강조 경로]: [정상 흐름(Happy Path)의 핵심 경로]
```

**Type A 품질 규칙:**
- 노드는 최대 10개로 제한. 초과 시 관련 단계를 하나의 노드로 그룹화
- 분기 조건은 반드시 한국어 라벨로 명시 (예: "성공 시" / "실패 시")
- 정상 경로(Happy Path)를 먼저 배치하고 별도 강조(스타일)로 구분
- 노드 텍스트는 짧은 동사구로 작성
- 복잡한 플로우는 "전체 흐름" + "상세 분기" 2개 다이어그램으로 분리
- 해당 문서 내 ```mermaid 코드블록으로 **인라인** 삽입

## Type B — 저해상도 HTML 와이어프레임

**목적:** 본 HTML 구현 전, 레이아웃·요소 배치를 저해상도로 먼저 확인한다.

**작성 규칙:**
- 그레이스케일만 사용 (회색 박스, 테두리, 플레이스홀더). 색·아이콘·실제 디자인 배제
- Tailwind 기준 예: `border border-gray-300 bg-gray-100 text-gray-500`
- 화면 명세서의 표시 데이터를 한국어 더미 데이터로 배치
- "미완성 컨셉"임이 드러나도록 유지 (실제 디자인처럼 보이지 않게)
- 상태별로 작성 (기본 / 빈 / 오류) — 상태별 파일 분리 또는 한 파일 내 상태 패널 병렬 배치 중 택1 (병렬 배치 시 파일명은 `-기본` 하나로 통일)

## Type C — 주석 HTML 도해 (Annotated)

**목적:** Type B 와이어프레임 위에 동작 규칙·조건·분기를 콜아웃으로 설명한다.

**작성 규칙:**
- 와이어프레임을 `position: relative` 컨테이너로 감싸고, 콜아웃을 `position: absolute`로 오버레이
- 콜아웃은 번호 뱃지(①②③) + 설명 텍스트 + 대상 요소를 가리키는 표시(선/화살표)로 구성
- 콜아웃 색상으로 의미 구분:

| 색상 | 의미 | Tailwind 예 |
|------|------|-------------|
| 빨강 | 오류·제약·비활성 조건 | `bg-red-500` |
| 파랑 | 사용자 액션·이동·교체 | `bg-blue-500` |
| 초록 | 활성·완료·정상 조건 | `bg-green-600` |
| 주황 | 경고·트리거·자동 감지 | `bg-amber-500` |
| 회색 | 기본 동작·닫힘 | `bg-gray-500` |

- 콜아웃 텍스트는 기획서의 동작 규칙 문구와 일치시킨다

## 출력 폴더 구조

```
docs/[프로젝트명]/
└── assets/
    └── wireframes/
        ├── wf-SC-AGENT-01-기본상태.html
        ├── wf-SC-AGENT-01-빈상태.html
        └── annotated-SC-AGENT-01-동작규칙.html
```

> Type A(플로우)는 별도 파일이 아니라 해당 문서 내 Mermaid 코드블록으로 인라인 포함한다.

## Phase 통합 지점

| Phase | 언제 작성 | 생성 유형 |
|-------|-----------|-----------|
| Phase 1 (PRD) | 사용자 플로우가 3단계 이상이거나 분기가 복잡한 경우 | Type A (Mermaid) |
| Phase 2 (UI 문서) | `02_user_flow.md` 작성 후 흐름 보완 필요 시 | Type A (Mermaid) |
| Phase 2 (UI 문서) | `03_screen_spec.md` 작성 후 레이아웃 컨셉 보완 필요 시 | Type B (와이어프레임) |
| Phase 3 (UI 생성) | 본 HTML 생성 직전, 화면별 레이아웃 확인 | Type B |
| Phase 3 (UI 생성) | UI 동작 규칙·조건·분기를 콜아웃으로 설명 필요 시 | Type C (주석 도해) |
