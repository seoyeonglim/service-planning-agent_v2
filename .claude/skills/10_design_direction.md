# Skill 10: 디자인 방향 (Design Direction)

## 목적
기획된 사용자 플로우와 화면을 일관된 시각 체계로 바인딩한다.

---

## 실행 프로토콜

### Step 1: 필수 정보 수집 질문

디자인 방향 수립 전 반드시 이 질문들을 해라:

1. "참고하고 싶은 서비스나 앱이 있나요?" → 레퍼런스 파악
2. "한 단어로 표현하면 어떤 느낌이길 원하시나요?" → 톤 파악
3. "주요 사용 환경은?" → 스마트폰 / 태블릿 / PC
4. "선호/회피 색상은?" → 컬러 파악

질문 없이 디자인 방향을 임의로 결정하지 말 것.

### Step 2: 시스템 값 정의

수집한 정보를 바탕으로 5가지 기본값을 결정한다.

- 컬러 팔레트(Primary/Secondary/Neutral/Danger)
- 타이포그래피(폰트, 제목/본문/캡션)
- 레이아웃 기준(모바일 우선, 최대 너비)
- 컴포넌트 스타일(버튼/카드/입력/배지)
- 톤 & 매너(아이콘/이미지/카피)

### Step 3: 결정 로그(ADR) 남기기

디자인 결정 시 대안 1개 이상 기록:

- 채택: 왜 이 톤인지
- 거절: 왜 다른 톤은 배제했는지
- 영향: 어떤 페르소나에 유리한지

---

## 범용 AICC 기본 디자인 방향 (참고용)

```markdown
## 디자인 방향: 범용 AICC

### 톤 & 매너
- 키워드: 정보 밀도 높고 신뢰감 있는, 장시간 봐도 피로 없는
- 참고 서비스: Zendesk, Freshdesk, Genesys (상담/업무 콘솔 계열)
- 피해야 할 느낌: 과하게 화려한, 채도 높은, 소비자용 마케팅 느낌

### 컬러 팔레트
| 용도 | 색상 | 헥스 코드 |
|------|------|-----------|
| Primary | 딥 블루 | #2563EB |
| Secondary | 라이트 블루 | #EFF6FF |
| Neutral | 차콜 | #1F2937 |
| Neutral Light | 오프 화이트 | #F8FAFC |
| Danger | 레드 | #E53E3E |
| Warning | 앰버 | #D97706 |
| Success | 그린 | #16A34A |

### 레이아웃 기준
- 기준 환경: 데스크톱 우선 (1280px, 상담 콘솔)
- 보조 대응: 태블릿 768px 이상
- 좌우 패딩: 24px
- 컴포넌트 간격: 8px 단위
```

### 생성 지시어(샘플)

- 색상, 타이포, 컴포넌트를 실제 클래스/변수로 변환해 UI 생성 문구로 추가
- 모든 화면에서 폰트 스케일과 여백은 일관성 있게 사용
- 한 화면 내에서 상태별 표시는 동일 규칙 사용

## 체크리스트

- 레퍼런스 반영 여부가 문서화되었는가?
- Primary 기준이 명확한가?
- 모바일/태블릿/PC 기준 중 1개가 확정되었는가?
- `요청사항 ID(REQ-###)`와 UI 샘플 지시가 연결되었는가?
- 각 페르소나의 사용성 조건과 충돌 여부 점검
