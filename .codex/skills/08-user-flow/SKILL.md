# Skill 08: 사용자 플로우 (User Flow)

## 목적
사용자가 목표를 달성하기까지의 화면 이동 경로를 정의한다.
스크린 설계와 테스트 케이스의 기초가 된다.

---

## 실행 프로토콜

### Step 1: 플로우 우선순위 결정

서비스 구조도 기반으로 플로우 우선순위를 정한다.

- 핵심 가치 전달 플로우 (MVP)
- 사용 빈도 높은 플로우
- 실패 시 임팩트 큰 플로우

각 플로우에 최소 하나 이상의 `REQ-###`를 연결한다.

### Step 2: 플로우 요소 정의

각 플로우는 다음 3가지를 포함:

1. **정상 경로**: 문제 없이 완료되는 경우
2. **분기 조건**: 조건별 분기
3. **예외 경로**: 실패/취소/권한 이슈 후 복구

또한 아래 항목을 명시:
- 시작/종료 액터
- 관련 화면(SC-###)
- REQ 매핑
- 완료 상태(성공 판정)
- 테스트 ID 연결(TC-###)

### Step 3: Mermaid 작성

반드시 Mermaid flowchart 사용.

- 사각형: 화면
- 마름모: 분기 조건
- 원형: 시작/종료
- 화살표 라벨: 사용자 액션 또는 시스템 조건

### Step 4: 승인 전 체크

플로우 문서 작성 후 다음 항목 미충족 시 다음 단계로 진행 불가:
- 모든 핵심 액션에 목적지/결과가 존재
- 예외 경로에서 사용자 복귀 지점이 명시
- 각 플로우에 최소 1개 에러 메시지 문안이 연결

---

## 플로우 산출물 포맷

```markdown
## 사용자 플로우

### 플로우 [번호]: [플로우명] (REQ-###)

- **액터:** [고객 / 상담사 / 상담관리자 / 시스템]
- **시작 지점:** [SC-ID]
- **종료 지점:** [SC-ID 또는 상태]
- **연관 화면:** [SC-ID 목록]
- **QA 연결:** TC-###, EC-###

**플로우 다이어그램:**
```mermaid
(다이어그램 내용)
```

**예외 케이스:**
| 상황 | 처리 방법 |
|------|-----------|
| [예외 상황] | [이동 / 메시지] |

**검증 시나리오:**
- Given: ...
- When: ...
- Then: ...
```

## 완성 체크리스트

- 서비스 구조도의 모든 화면이 최소 1개 플로우에 등장하는가?
- 모든 플로우에 정상/예외 경로가 모두 있는가?
- 모든 분기점에 조건이 명시되어 있는가?
- 플로우 종료 후 사용자가 어디로 돌아가는지 명시되었는가?
- 각 플로우에 REQ-###와 TC-###가 연결되었는가?
